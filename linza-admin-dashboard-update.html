Auth Flow (OTP-совместимый)

1. Login → POST /api/auth { login, password }
   ↓ Проверка credentials (bcrypt) → Генерация 6-значного OTP → Сохранение в БД
   ↓ Response: { user: { id, email, phoneNumber }, stateToken }

2. OTP Challenge → POST /api/auth/factors/otp/challenge/email { stateToken }
   ↓ Перегенерация OTP, отправка по email/sms

3. OTP Verify → POST /api/auth/factors/otp/verify { stateToken, passcode }
   ↓ Проверка OTP (bcrypt verify) → Отметка verified=true
   ↓ Response: { user, stateToken }

4. Sign In → POST /api/auth/sign-in { stateToken }
   ↓ Генерация JWT: { sub, role, pr, ar, tid, gid, exp }
   ↓ Response: { accessToken } → localStorage

5. Portal Role Switch → POST /api/auth/switch-role { active_role }
   ↓ Новый JWT с обновлённым ar (active_role) полем
