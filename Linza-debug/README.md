# Linza Debug

Сервис централизованного отслеживания и управления ошибками экосистемы **Linza Detector**.

## Описание

Linza Debug — внутренний сервис для сбора, классификации и отслеживания ошибок, возникающих в микросервисах платформы Linza Detector. Ошибки фиксируются автоматически (через middleware в каждом сервисе) и вручную (через интерфейс Linza Board), а затем могут быть отправлены в [GitHub Issues](https://github.com/BigDataQueen/Linza-debug/issues) для дальнейшей работы команды.

## Отслеживаемые сервисы

| Сервис | Описание | Типы ошибок |
|--------|----------|-------------|
| `linza-board` | Клиентское приложение (Vue.js + FastAPI) | UI-ошибки, ошибки API, авторизации |
| `linza-vpleer` | Видеоплеер с детекциями | Ошибки воспроизведения, рендеринга детекций |
| `linza-storage-service` | Сервис работы с S3-хранилищем | Ошибки загрузки/выгрузки файлов, подключения к S3 |
| `linza-analytics` | Сервис аналитики | Ошибки обработки данных, генерации отчётов |

## Классификация ошибок

### По серьёзности (severity)

| Уровень | Описание |
|---------|----------|
| `critical` | Необработанные исключения, падение сервиса |
| `error` | HTTP 5xx ответы, ошибки бизнес-логики |
| `warning` | HTTP 4xx ответы, некорректные запросы |
| `info` | Информационные события для отладки |

### По категории

| Категория | Описание | Примеры |
|-----------|----------|--------|
| `ui` | Ошибки интерфейса | Не отображается раздел, сломана навигация |
| `api` | Ошибки серверного API | 500 на эндпоинте, таймаут запроса |
| `storage` | Ошибки хранилища | Не удаётся загрузить файл, ошибка S3 |
| `auth` | Ошибки авторизации | Невалидный токен, отказ в доступе |
| `player` | Ошибки видеоплеера | Не воспроизводится видео, ошибка детекций |
| `analytics` | Ошибки аналитики | Не загружается отчёт, ошибка фильтрации |
| `network` | Сетевые ошибки | Таймаут соединения, недоступен сервис |

> **Автоматическая категоризация:** Если `category` не указана в отчёте, определяется
> по `service`: vpleer→player, storage-service→storage, analytics-service→analytics,
> остальные→api. При явном указании используется переданное значение.

### Типовые ошибки по сервисам

#### linza-board
- Ошибки рендеринга Vue-компонентов
- Сбои авторизации (JWT expired, invalid token)
- Ошибки CRUD операций (пользователи, хранилища, источники)
- Ошибки загрузки и парсинга JSON-отчётов

#### linza-vpleer
- Ошибки воспроизведения видео (кодеки, формат)
- Сбои отображения детекций (base64, overlay)
- Ошибки iframe-интеграции

#### linza-storage-service
- Ошибки подключения к S3 (credentials, endpoint)
- Сбои загрузки/выгрузки файлов
- Таймауты при работе с большими файлами

#### linza-analytics
- Ошибки генерации аналитических отчётов
- Сбои при фильтрации и классификации данных

## API

### POST /api/errors/report

Принимает отчёт об ошибке от любого сервиса.

**Request body:**
```json
{
  "service": "linza-board",
  "severity": "error",
  "message": "Описание ошибки",
  "category": "ui",
  "traceback": "...",
  "endpoint": "/api/reports",
  "method": "GET",
  "status_code": 500,
  "request_id": "abc123",
  "extra": "дополнительные данные"
}
```

### GET /api/errors

Список всех зафиксированных ошибок с фильтрацией.

**Query params:** `service`, `severity`, `category`, `status`, `limit`, `offset`

### POST /api/errors/{id}/submit-issue

Отправка ошибки как GitHub Issue в репозиторий Linza-debug.

## Переменные окружения

| Переменная | Описание | По умолчанию |
|-----------|----------|--------------|
| `GITHUB_TOKEN` | Токен для создания GitHub Issues | — |

## Интеграция с Linza Board

В интерфейсе Linza Board доступен раздел **Настройки → Отслеживание ошибок**, который позволяет:

- Просматривать все зафиксированные ошибки с фильтрацией по сервису, категории и серьёзности
- Отправлять ошибки как задачи в GitHub Issues репозитория Linza-debug
- Отслеживать статус отправленных задач
