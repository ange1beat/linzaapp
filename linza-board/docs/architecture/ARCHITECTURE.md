# Linza Detector — Архитектура

## Обзор

Linza Detector разделена на независимые сервисы, каждый в своём репозитории:

| Репозиторий | Назначение | Порт |
|-------------|-----------|------|
| **linza-board** | UI (Vue.js SPA), авторизация, роли пользователей | 8000 |
| **Linza-storage-service** | Работа с файлами, S3-хранилище, управление доступом | 8001 |
| **linza-analytics** | Анализ видео, классификатор нарушений, отчёты | 8002 |
| **linza-vpleer** | Видеоплеер, воспроизведение и навигация по видео | 8003 |
| **Linza-deploy** | Сборка единого Docker-контейнера для развёртывания | — |

## Архитектурная схема

```
┌───────────────────────────────────────────────────────────┐
│                    Nginx (API Gateway)                     │
│                   Linza-deploy контейнер                   │
└──────┬────────────┬────────────────┬──────────┬───────────┘
       │            │                │          │
┌──────▼──────┐ ┌───▼──────────┐ ┌──▼────────┐ ┌▼──────────┐
│ linza-board │ │ storage-     │ │ analytics │ │ vpleer    │
│             │ │ service      │ │           │ │           │
│ Vue.js SPA  │ │ S3/MinIO     │ │ Classifier│ │ Video     │
│ Auth/JWT    │ │ Файлы        │ │ Отчёты    │ │ Player    │
│ Роли        │ │ Доступ       │ │ Excel     │ │           │
└─────────────┘ └──────────────┘ └───────────┘ └───────────┘
```

## Стек технологий

| Компонент | Технология |
|-----------|-----------|
| Фронтенд | Vue.js 3, Vite, Vue Router |
| Бэкенд | Python 3.12, FastAPI |
| Хранилище | MinIO (S3-совместимое) |
| БД | SQLite (аналитика), PostgreSQL (сервисы) |
| Контейнеризация | Docker, Docker Compose |
| API Gateway | Nginx |

## Взаимодействие

- Фронтенд (linza-board) обращается к сервисам через API Gateway
- В режиме разработки Vite проксирует запросы к локальным сервисам
- В продакшене Nginx маршрутизирует запросы по URL-префиксам
